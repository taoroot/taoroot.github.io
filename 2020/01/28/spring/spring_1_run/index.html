MyApplication

spring boot 简单的配置即可启动程序。

静态方法run主要是为了创建SpringApplication对象，并且调用SpringApplication的另一个run方法。

mainApplicationClass

主函数类

顾名思义，就是main方法所在的类。利用RuntimeException找到main方法是哪个类里面。这里要做区分。并不是把run方法的传参primarySources为mainApplicationClass，虽然这里都是同一个。

run()

先来看看总流程

SpringApplicationRunListener

顾名思义，该监听器是监听在SpringApplication的run方法中的，在run方法执行的不同的阶段触发不同的事件。

SpringApplicationRunListeners类是有一个SpringApplicationRunListener集合属性listeners, 和一个log属性。很容易就猜出，这是负责批量处理SpringApplicationRunListener的。

不过springboot只配置了一个SpringApplicationRunListener实现类，即EventPublishingRunListener。

ApplicationContext 刷新

这里面就是Spring的核心流程，之前的所有操作都是准备操作。
在这里将涉及到IOC，web容器的是初始化。内容过多，需要在下一篇文章再讲。

总结

Spring boot的启动流程，就是创建一个SpringApplication并调用run方法的过程。

构造器：

  1. 属性primarySources可以是任意的class，至少一个，我们这里是MyApplication.class，其实是需要有@SpringBootApplication即可。本文没体现出来，下一篇文章分析。
  2. 属性webApplicationType 判断web类型。
  3. 属性initializers 6个初始化器。用于初始化容器
  4. 属性listeners 10个监听器。

执行run：

  1. 一个计时器 StopWatch
  2. SpringApplicationRunListeners批量处理SpringApplicationRunListener
  3. SpringApplicationRunListener用于在执行run方法的不同阶段，通过不同的事件触发不同的监听器listeners。并且在容器初始化后，讲listeners设置为容器的监听器。
  4. 通过webApplicationType 创建不同的容器ConfigurableApplicationContext，环境ConfigurableEnvironment。
  5. initializers 在容器创建好以后开始干活。
  6. 容器刷新refresh。
  7. 执行ApplicationRunner和CommandLineRunner。
