Java I/O系统

发表于 2020-02-13 更新于 2020-02-14 人数：

流

io系统即输入输出系统。

概述

Java 对象通过流来完成输出和输出。流是生产和消费信息的抽象。流通过Java的输入和输出系统与物理设备连接。尽管与他们连接的物理设备不尽相同，但所有流的行为具有相同的方式。因此，相同的输入/输出类和方法适用于所有类型的外部设备。这以为这一个输出输入流能够抽象多种不同类型输入；比如，磁盘我呢见，从键盘，从网络套接字。同样，一个输出流也可以输出到控制台，磁盘，或者相连网络。流是处理输入/输出的一个洁净的方式。例如他不需要代码理解和网络的不同。Java中流的实现是在java.io包定义的类层次结构内部的。

分类

从功能上分，流分为输出流和输入流。这是以应用程序作为参考系，如果从应用程序从外部读入数据，那么这就是输入流，如果是写出数据，那么就是输出流。
从流的结构上分，可分为字节流和字符流。字节流按照字节处理数据，字符流按照字符处理数据。

字节流的输入和输出基础是InputStream/OutputStream这两个抽象类，字节流的输入和输出操作由这两个类的子类实现，因为java不同同时继承两个抽象类，所以所有流，不可能同时是输入又是输出流。
字符流是Java 1.1以后新增的以字符作为单位进行输入和输出处理的流。字符流的的输入和输出基础是Reader和Writer这两个接口。不过要注意，在最底层，所有的输入和输出都是字节形式的。基于字符的流只为了处理字符提供方便有效的方法。

从流的读写对象可分为节点流和过滤流，节点流的读写对象是从特定设备读取。而过滤流的读写对象是其他流。 节点流 —> 过滤流 —> 过滤了 —> 节点流。过滤流起到对流的包装，增强流的功能。也就是设计模式中的装饰者模式。

装饰模式Decorator

概念

  1. 装饰模式又称包装模式(wrapper)
  2. 装饰模式以对客户端透明的方式扩展对象的功能，是继承关系的一个替代方案。
  3. 装饰模式以对客户端透明的方式动态的给一个对象附加更多的责任。换言之，客户端并不会觉得对象在装饰器和转世后有所不同。
  4. 装饰模式可以在不创建更多子类的情况下，讲对象的功能进行扩张。
  5. 装饰模式吧客户端的调用委派到被装饰类。装饰模式的关键在于这种扩展完全透明
  6. 转世模式是在不必改变原来文件和使用继承的情况下动态的扩展一个对象的功能。他是通过创建一个包装对象，也就是装饰来包裹真是的对象。

角色

抽象构件角色(Component): 给出一个抽象接口，以规范的准备接收附加责任的对象。

具体构件角色(Concrete Component): 定义一个将要接收附加责任的类。

装饰角色(Decorator): 持有一个构件，对象的应用，并定义要给与抽象接口一致的接口。

具体装饰角色(Concrete Component)：负责给构件对象”贴上”附加的责任。

特点

  1. 装饰对象和真是对象有相同的接口。这样客户端对象可以以和真是对象相同的方式和装饰对象交互。
  2. 解释对象包含一个真是对的引用
  3. 转世独享接收来自客户端的请求。吧这些请求转发真是的对象。
  4. 装饰对象可以在转发这些请求以前或者以后附加一些附加功能。这些就确保了运行时，不用修改给定对象的结构就可以在外部附加功能。通常时通过继承来实现给定类的功能扩展.
