JVM 解读

发表于 2019-11-30 更新于 2019-12-01 阅读次数：

JVM是Java Virtual Machine（Java虚拟机）的缩写，JVM是一种用于计算设备的规范，它是一个虚构出来的计算机，是通过在实际的计算机上仿真模拟各种计算机功能来实现的。
引入Java语言虚拟机后，Java语言在不同平台上运行时不需要重新编译。Java语言使用Java虚拟机屏蔽了与具体平台相关的信息，使得Java语言编译程序只需生成在Java虚拟机上运行的目标代码（字节码），就可以在多种平台上不加修改地运行。

操作系统架构

要想了解JVM，得想了解一下，软件是怎么跑起来的。采购过组装机的都有经验，要买CPU，内存，主板，硬盘，以及其他外设。 CPU 就是计算中心了。内存，磁盘都是存储设备，不过内存断点数据就消失了，磁盘是不会消失。 我们来讲一下程序是怎么跑起来的。程序就是存储在磁盘上的文件上一个文件。当我们打开程序的时候，文件就被拷贝到内存里，变成了一个进程。进程是什么呢？就是把程序加载以后，可以被CPU可执行的东西。开机的时候，电脑就是会先加载操作系统到内存，让CPU执行，那也意味着在内存中的操作系统也是一个进程。

进程 ？ 线程

搜索一下解释 百度百科

进程是操作系统分配资源的基本单位

线程是CPU调度的基本单位

按照解释，在启动程序的时候，操作系统会先分配资源，然后加载程序，成为一个进程，创建线程，最后CPU调度

Linux 进程调度

在Linux系统里，线程是一个特殊的进程，与进程共享内存地址以及打开的文件等等。

java编译执行原理图

由于在不同的CPU的执行指令集不一样， 也就意味着如果要用C语言编写程序，就得用不同平台的编译器编译一次，才可以生成对应平台所能执行的程序。而JVM就底层屏蔽了这一点，java生成的字节码可以加载到不同平台的JVM中，翻译解释为对应平台的指令集，也就是所谓跨平台【一次编译，到处执行】。
JVM有堆内内存，和堆外内存。JVM本身就是一个进程，也就是，将系统分配的给JVM的堆内存中的一部分，一共给了Java线程，也就是堆内内存。所谓的对外内存，就是出了这些分配的内存，结合上一章内存进程布局图。

查看字节码

JVM 是C++编写的，当我们用java启动线程的时候，肯定是通过C++去执行的。也就是thread.start() 会去调用C++的方法，创建一个系统线程，java采用native关键字表示，当前方法是C++的方法。创建了线程就可以执行java了，就是run() 里面的东西。
所以Runnable就是需要执行的代码，Thread就是存储了对应JVM创建的系统线程的信息。
