发表于 2019-11-22 更新于 2019-11-25 阅读次数：

MyBatis 是一款优秀的持久层框架，它支持定制化 SQL、存储过程以及高级映射。MyBatis 避免了几乎所有的 JDBC 代码和手动设置参数以及获取结果集。MyBatis 可以使用简单的 XML 或注解来配置和映射原生类型、接口和 Java 的 POJO（Plain Old Java Objects，普通老式 Java 对象）为数据库中的记录。

mybatis 动态代理 Mappper

我明明只指定义了BlogMapper selectBlog 接口，没有具体的实现类，为什么就可以使用了呢？

关键字： jdk动态代理 Proxy.newProxyInstance

一个简单的demo，动态代理解析

要跑这个demo 需要在pom.xml 里面配置一下 javac 参数， 不然就拿不到参数名称，就会变成 arg0， arg1了， 也就是为什么mybatis多个参数的时候需要使用@params

这个简单的demo只是实现了mybatis功能中的一步，生成sql，mybatis干的事件比这个多多了。

源码

目录

看源码就从入口找起，也就是 SqlSessionFactory 了。

SqlSessionFactory

创建sqlsession，从外部的connection或是datasource。mybatis是把jdbc的一次标准流程操作抽象成了一个sqlsession。当我们需要用的时候，利用就是工程打开一个session，进行CRUD，不用再去管connection， statement

Sqlsession

对于使用者，只需要关心如何使用sqlsession就行了

SqlSessionFactoryBuilder

要想获取 SqlSessionFactory， 官网是通过SqlSessionFactoryBuilder提供了两种方式。
第一种是读取xml配置文件，另一种直接用代码new Configuration出来。
其实第一种就是对第二种的封装，通过读取配置文件，mybatis自己去new configuration对象。
从源码中可以看出， 最终都是走了SqlSessionFactory build(Configuration config) 方法
关于mybatis是解析xml就不做分析了，是调用了jdk 自带的 xpath。

DefaultSqlSessionFactory

源码跟踪到这里，发现走到地了。 DefaultSqlSessionFactory 只是赋值了一下，并没有做啥操作。
既然build 方法没做奇怪的操作，说明在configfigure创建的时候就以及完成了。也就是configuration.addMapper(BlogMapper.class);

Configuration

这个类在mybatis经常出现，而且代码里面包含了很多对象。以后用到，在慢慢加回来

MapperRegistry

顾名思义，这个类主要就是用来管理mapper了

MapperProxyFactory

保存 原始 mapper

MapperAnnotationBuilder

绑定 xml和mapper，加载maper.xml
这里也说明了xml默认加载的地址是mapper的径下的同名.xml 文件
