threejs 导入FBX大模型解决思路

发表于 2019-10-23 阅读次数：

我只是提供一个思路，解决工作中问题而已，代码就那样，结果才是最重要，
把所有重复的模型去掉以后，900M FBX模型变成了 45M json + 10M 图片

模型师是用3dmax建模，导出的场景就有900M，这里的fbx包括了贴图，所以大很多，
threejs 提供的 fbx2tothreejs 脚本，没法直接使用， 有两点原因，nodejs 字符串长度有限制，
没法对有图片的fbx进行导出， 想想看 fbx都900M了， 导出的json估计都有2G。没法用。

其实fbx模型文件是有很多重复的， 解决办法就是手动去重

思路：

  1. fbx 文件太大？

    对模型内容分组，重复的放到一块， 按组导出， 001.fbx 002.fbx 003.fbx ….

  2. 带图片fbx 无法转 threejs json ？

    看了下导出脚本就知道了， 脚本里面用到了 blob， 这个是nodejs没有的，浏览器所以可以在 https://threejs.org/editor/ 导入，然后再导出 json， 这里选择 导出物体（而不是场景!!!!)

  3. 去重

    对每一个fbx去重， 直接看代码把。

批处理， 上面的脚本，每次只能一个转json，所以来一个批处理脚本

  1. 合并模型
    threejs editor 上导入所有json （CTRL + A 全选），等加载完以后， 选择导出
    场景（这里要场景！！！）

  2. 再去重

合并以后还是有重复的， 而且里面也有图片 （base64着），这里就没必要保存json里面了

最终会生成 scene.json （40M） + img 文件夹（里面就是贴图）
完..
