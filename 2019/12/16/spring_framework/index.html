Spring Framework 浅析

发表于 2019-12-16 更新于 2019-12-18 阅读次数：
本文字数： 3.3k 阅读时长 ≈ 3 分钟

Spring框架是由于软件开发的复杂性而创建的。Spring使用的是基本的JavaBean来完成以前只可能由EJB完成的事情。然而，Spring的用途不仅仅限于服务器端的开发。从简单性、可测试性和松耦合性角度而言，绝大部分Java应用都可以从Spring中受益。

https://docs.spring.io/spring/docs/5.2.1.RELEASE/spring-framework-reference/core.html

spring-context | IOC

如果项目中使用了Spring, 是不是意味着开发者不需要再去new object()，使用的过程中，只需要使用接口类即可， 这是因为Spring帮我们动态代理着，这个是JDK提供的方法。如果像知道什么动态代理，可以移步到另一篇文章mybatis解读 mybatis 动态代理 Mappper 一节。 spring 把这一行为定义为IOC，就是平时说的控制反转，本来应该由开发者去手动new的，改由框架自己干了。那到底是什么完成的呢。 如果要让框架自己完成实现类的注入，那前提是得告诉框架应该在哪里找，就是xml配置了。

spring-mvc

https://docs.spring.io/spring/docs/5.2.1.RELEASE/spring-framework-reference/web.html#mvc

Spring Web MVC是在Servlet API上构建的原始Web框架，从一开始就包含在Spring框架中。其正式名称“Spring Web MVC”来自其源模块(Spring-webmvc)的名称，但它更常见的名称是“Spring MVC”。

启动流程分析

spring-mvc 是放在servlet容器里面跑的，例如tomcat。为什么需要这个容器，因为你需要有人帮你管理连接。
创建一个servlet，并且当外部访问这个servlet的时候，会转发到spring-mvc的前端路由上，就是赋值管理controller list，从而去执行service里面的方法。

servlet可以通过xml文件，以及class文件创建

根据 class 启动

这种是在 根目录下创建 /META-INF/services/javax.servlet.ServletContainerInitializer ，一行写一个class的classpath。

可以看一下spring-web.jar 的。spring以及默认帮我们写了一个实例。不需要我们手动再去创建了。

SpringServletContainerInitializer

WebInitializer

所以我们只需要在代码中创建一个类，并且实现 SpringServletContainerInitializer接口就可以。

spring 架构很明确。 spring-mvc 创建的容器主要是管理controller一层的， 像dao，service这种，还是按照以前的方式注入。真的完全解耦了。
等 ContextLoaderListener 把 spring 容器创建完，只需要setparent 一下，在controller层就可以用dao，service的代理类了。
